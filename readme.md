# Business-Driven-AI

取名参考的是《Domain-Driven-Development》，也就是著名的《领域驱动设计》。随着 AI 工具的引入，我们发现代码数量上去了，但是代码质量却是急转直下。这里的代码质量不是代码本身写的漂不漂亮，而是实现的内容，难以让人理解其背后的业务背景。我们需要回归软件开发的本源，从明确业务、定义上下文开始，重新思考我们对 ai 工具的使用方式。所以是《业务驱动 ai》
